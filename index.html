Identity & Access Management (IAM)

Module 2
IAM = Security Foundation: Every OCI API call is authenticated and authorised by IAM. No resource can be created, modified or deleted without a policy explicitly allowing it. Getting IAM right is the #1 skill for OCI architects.

🔐 IAM Components

UsersHuman or service accounts
GroupsCollection of users
PoliciesAllow statements on resources
CompartmentsLogical resource containers
Instance PrincipalsCompute calling OCI APIs
FederationSAML/IDCS identity sync

📋 Policy Syntax

Allowallow group X to verb resource
Verbsinspect/read/use/manage
Resourcesall-resources/instances/vcns...
Locationin tenancy / in compartment X
Conditionswhere request.region=...
TaggingDefined + Free-form tags

Networking & VCN

Module 3
VCN = Your Private Network in OCI: Every compute instance, database, and load balancer lives inside a VCN. Designing VCN correctly — subnets, gateways, security lists — is the foundation of every OCI deployment.
🌐Networking Components
  • 🔵
    VCN & CIDR — Virtual Cloud Network with IP range (e.g. 10.0.0.0/16). Subdivided into subnets.
  • 🔵
    Subnets — Public (internet-accessible) vs Private (internal only). Regional subnets span all ADs.
  • 🔵
    Internet Gateway (IGW) — Enables public subnet resources to communicate with internet.
  • 🔵
    NAT Gateway — Allows private subnet to initiate outbound internet connections without exposing IP.
  • 🔵
    Dynamic Routing Gateway (DRG) — Connects VCN to on-premises network via VPN or FastConnect.
  • 🔵
    Service Gateway — Private access to Oracle services (Object Storage) without internet.
  • 🔵
    Security Lists vs NSGs — Stateful vs Stateless rules. NSG = instance-level, Security List = subnet-level.
  • 🔵
    VPN Connect + FastConnect — IPSec VPN (internet) vs FastConnect (dedicated private circuit, lower latency).

Block, Object & File Storage

Module 5
3 Storage Types in OCI: Block Storage (like a hard disk for VMs), Object Storage (like S3 — for files, backups, images), File Storage (shared NFS — for multiple instances). Each has different use cases, performance, and pricing.

💿 Block Storage

Boot VolumeOS disk — auto-created with instance
Block VolumeAdditional disk — attach/detach
AttachmentiSCSI or Paravirtualized
PerformanceLower/Balanced/Higher/Ultra
BackupManual or scheduled policy
CloneInstant copy of volume

🪣 Object Storage

Standard TierFrequently accessed data
Infrequent AccessLower cost, min 31 days
Archive TierLowest cost, restore needed
Pre-Auth RequestTime-limited public URL
Lifecycle PolicyAuto-move/delete objects
VersioningKeep multiple object versions

📁 File Storage

ProtocolNFS v3 — shared file system
Use CasesShared home dirs, app data
Mount TargetNFS endpoint in your VCN
SecurityKerberos, Unix permissions
SnapshotsPoint-in-time file system copy
ReplicationCross-region file replication

Managing Databases on OCI

Module 6
OCI Database Service: Oracle offers 3 deployment options — VM DB Systems (flexible, cost-effective), Bare Metal DB Systems (high performance, dedicated), and Exadata DB Systems (maximum performance). All managed by Oracle.
🗄️Database Topics Covered
  • 🔵
    VM DB Systems — Flexible VM-based Oracle DB. 1-2 node RAC. Good for dev/test and mid-tier production.
  • 🔵
    Bare Metal DB Systems — Dedicated physical server. X7/X8 shapes. Highest IOPS for single-tenant workloads.
  • 🔵
    Exadata DB Systems on OCI — Full Exadata stack in OCI. X7 systems. Smart Scan, Flash Cache, IORM.
  • 🔵
    Database Editions + Versions — SE2, EE, EE-HP, EE-EP. Oracle DB 11g, 12c, 19c, 21c.
  • 🔵
    Patching — DB System patching via OCI Console. Quarterly patches, rolling vs non-rolling.
  • 🔵
    Backup + Restore — Automatic backups to Object Storage. Manual on-demand backup. RMAN restore.
  • 🔵
    Data Guard Standby — Automatic failover, sync/async redo transport, Active Data Guard for read queries.
  • 🔵
    Migration — Lift & Shift — Move on-prem Oracle DB to OCI using Data Pump, RMAN, Zero Downtime Migration.
  • 🔵
    Pricing — BYOL — Bring Your Own License: use existing on-prem Oracle licenses in OCI. Significant cost saving.

Q1. Which OCI gateway allows instances in a private subnet to initiate outbound internet connections WITHOUT exposing their private IP?

A
Internet Gateway (IGW)
B
NAT Gateway
C
Dynamic Routing Gateway (DRG)
D
Service Gateway
NAT Gateway = Network Address Translation. Private subnet instances use NAT Gateway to initiate outbound-only internet connections (e.g. OS updates) without getting a public IP. IGW = for public subnets with public IPs. DRG = on-premises connectivity. Service Gateway = Oracle services only.

Q2. In OCI IAM, which statement correctly represents the policy verb hierarchy from least to most privileged?

A
use → read → inspect → manage
B
inspect → read → use → manage
C
read → inspect → manage → use
D
inspect → use → manage → read
inspect (list resources) → read (inspect + get) → use (read + create/update some) → manage (full control including delete). This is a classic 1Z0-1085 question — memorise this order.

Q3. Which Compute shape type is specifically designed for memory-intensive workloads like in-memory databases with local NVMe SSDs?

A
Standard Shape
B
GPU Shape
C
DenseIO Shape
D
HighIO Shape
DenseIO shapes have large amounts of local NVMe SSD storage — ideal for in-memory databases (like SAP HANA) and big data workloads. HighIO = optimised for OLTP databases (high IOPS block storage). GPU = AI/ML/HPC. Standard = general purpose.

Q4. What is the correct sequence for Exadata patching on OCI?

A
DB → GI → Cell → Switch
B
GI → DB → Cell → Switch
C
Cell → DB Node OS → GI → DB
D
Switch → Cell → GI → DB
Exadata patching order: Cell servers first → DB Node OS → Grid Infrastructure (GI) → Database software. Cell servers are patched first because all I/O flows through them. GI must be patched before DB. This sequence prevents data corruption and ensures rolling patch success.

Q5. Which OCI Object Storage tier should you choose for data that is accessed less than once a month and must be retained for compliance?

A
Standard Tier
B
Infrequent Access Tier
C
Archive Tier
D
Block Storage
Archive Tier = lowest cost, for rarely accessed data (compliance archives, cold backups). Objects need to be restored before access (restore takes hours). Infrequent Access = moderate cost, min 31-day retention, quicker access. Standard = frequent access, highest cost per GB.
